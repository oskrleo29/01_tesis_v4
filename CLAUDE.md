# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

Master's thesis (Universidad Central, 2026) on estimating personal spending (`gasto personal`) as a proxy for credit capacity, using ENPH 2016-2017 microdata (DANE), CRISP-DM methodology, WLS regression weighted by expansion factor `FEX_C`, and K-Means clustering with a logistic production classifier.

## Running the Streamlit app

From the project root:

```bash
cd 02_app && streamlit run app.py
```

On Windows: double-click `02_app/run_streamlit.bat`.

**Prerequisite**: The artifact `artefactos_modelo_hibrido.pkl` must exist at the project root. It is generated by running the hybrid model cells in the notebook (K-Means training + logistic classifier fitting + artifact export).

## Architecture

### Notebook (`Proyecto_gasto_personal_2026_tesis_HIBRIDO_v6.ipynb`)

The main notebook follows CRISP-DM:

| Section | Cells | Purpose |
| ------- | ----- | ------- |
| 1. Data ingestion | 6–8 | Downloads ENPH microdata from Google Drive via `gdown`, loads cuadernillos |
| 2. Cleaning | 9–35 | Homologation, income construction (`ingreso_empleado`, `ingreso_independiente`, `Pension`), subsidy adjustments |
| 3. Dataset integration | 74–86 | Merges expense sources (gastos diarios, menos frecuentes, personales, comidas), indexes to 2025 prices, builds `df_temp_a` |
| 4. Descriptive analysis | 87–107 | FEX_C-weighted statistics, divergence tables (muestra vs. expandido) |
| 5. Variable selection | 108–110 | Kruskal-Wallis, weighted Spearman/Pearson, VIF, Cramér's V |
| 6. WLS regression | 112–116 | Fits `log_gastos_2025 ~ f(variables)` with `FEX_C` weights; builds `log_gasto_pred_w` and `residuo_log_gasto` |
| 7. K-Means clustering | 117–120 | Weighted K-Means (k=4) on residuals + socioeconomic features; produces `cluster_ord_w` |
| 8. Hybrid model | 121–125 | Trains logistic classifier on cluster labels; exports `artefactos_modelo_hibrido.pkl` |

The three standalone `.py` scripts are **not standalone executables** — they are called from the notebook with `exec(open('script_name.py').read())` after `df_temp_a` is loaded:

- `script_descriptivos_ponderados.py` — weighted descriptive tables
- `script_correlaciones_y_boxplots.py` — correlation matrices and boxplots
- `script_seleccion_variables.py` — variable selection tests and export

### Production app (`02_app/`)

```text
pipeline.py        ← Inference engine (the core logic)
business_rules.py  ← All domain mappings and cluster profile definitions
app.py             ← Streamlit UI (calls pipeline.predict())
persistence.py     ← Appends each inference to resultados_inferencia.csv
```

**Inference flow in `pipeline.py`** (v3 — hybrid model):

1. Map raw form inputs → WLS model features via `_construir_df_regresion()`
2. `model_wls.predict()` → `log_gasto_pred` → `gasto_estimado = exp(log_gasto_pred)`
3. `ratio_estimado = gasto_estimado / ingreso`; `log_ratio_estimado = log(ratio_estimado)`
4. Map inputs + estimated values → classifier features via `_construir_df_clf()`
5. `clf_pipeline.predict()` → `cluster_ord` (0–3); `clf_pipeline.predict_proba()` → `proba_c3`
6. `senal_alta_presion = proba_c3 >= UMBRAL_C3` (threshold stored in artifact)
7. `RATIO_TABLE[cluster_ord]` → percentile position using `p25_obs/p50_obs/p75_obs/p90_obs` → risk classification

### Artifact structure (`artefactos_modelo_hibrido.pkl`)

A dict with keys: `model_wls`, `clf_pipeline`, `ratio_table_hibrido`, `cluster_map_hibrido`, `umbral_c3_optimo`.

- `ratio_table_hibrido`: `{cluster_ord: {p25_obs, p50_obs, p75_obs, p90_obs, residuo_mediano}}` — empirical percentile thresholds per cluster using **observed** ratios
- `cluster_map_hibrido`: `{cluster_raw_int: cluster_ord_int}` — maps K-Means internal label to interpretable rank (0 = lowest ratio, 3 = highest)
- `umbral_c3_optimo`: optimal probability threshold for flagging high financial pressure (default 0.50)

The artifact is loaded at module import time in `pipeline.py` (not lazily). `_validar_artefactos()` checks for all required keys on load.

### Key data variables (notebook context)

| Variable | Description |
| -------- | ----------- |
| `df_temp_a` | Main modeling DataFrame (aportantes del hogar, public objetivo) |
| `FEX_C` | Expansion factor — must be used as `weights=` in all statistical operations |
| `log_ingresos_2025` | `log(ingreso_mensual_total)` indexed to Dec 2025 |
| `log_gastos_2025` | `log(GASTOS_AL_2025)` — the regression target |
| `log_gasto_pred_w` | WLS model predictions (log scale) |
| `residuo_log_gasto` | WLS residuals — used as K-Means clustering input |
| `cluster_ord_w` | K-Means cluster assignment (0–3, ordered by ratio) — training labels for logistic classifier |
| `SMMLV_2025` | 1,300,000 COP — minimum wage reference for 2025 |

### Business rules (`business_rules.py`)

All categorical domain mappings live here (not in `pipeline.py`): `MAP_CIUDAD`, `MAP_NIVEL_EDUCATIVO`, `MAP_ACTIVIDAD`, `MAP_TIPO_VIVIENDA`, `MAP_ANTIGUEDAD`, `MAP_GRUPO_EDAD`, `MAP_GRUPO_APORTANTES`, and `CLUSTER_PROFILES`. When the model is retrained with new categories, update these mappings first.

`MAP_CIUDAD` maps city names (uppercase) to `{dominio_agrup, REGION}`. Cities not in this dict cause a `ValueError` at inference time.

`clasificar_riesgo(percentil)` maps `"p25"/"p50"/"p75"/"p90"/"p100"` → `{nivel, color, descripcion}`.
